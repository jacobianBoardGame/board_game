项目背景

籌碼阿瓦隆是一个基于阿瓦隆桌游体验而逐步形成的筹码规则扩展与结算系统。 它的目标不是替代原游戏，而是在社交局、熟人局或内容展示场景中， 提供更明确的风险、奖励、旁盘与局后结算结构。

当前站点采用 GitHub Pages 静态部署，方便低成本公开、持续更新与版本记录。 站点结构被拆分为首页、规则介绍、完整规则、结算工具、FAQ、更新记录与本页面， 以便后续扩展而不把所有内容塞进单一页面。

Project Background

Chip Avalon is a chip-based rule extension and settlement system developed around the social and replayable experience of Avalon-style gameplay. Its purpose is not to replace the original game, but to provide clearer incentives, side markets, and post-game settlement structures for friendly games and online presentation.

The current site is deployed with GitHub Pages for low-cost public publishing, continuous revision, and visible version history. The content is split across multiple pages so the project can grow without forcing everything into a single long page.

项目时间线

阶段 1：规则形成

从社交局实践中形成主局分、旁盘、奖池与结算逻辑的基础框架。

阶段 2：文本整理

将零散规则整理为可公开阅读的页面，并开始形成版本意识与结构化目录。

阶段 3：站点拆分

将项目拆分为首页、规则页、快速介绍页与结算页，为未来扩展预留空间。

阶段 4：持续更新

后续可继续加入 FAQ、边界情况、示例局、赔率变体与更完整的结算工具。

Project Timeline

Stage 1: Rule Formation

The core structure emerged from actual social sessions, including base scores, side markets, prize pools, and settlement logic.

Stage 2: Rule Documentation

Scattered notes were organized into public-facing text with clearer structure and version awareness.

Stage 3: Site Split

The project was split into homepage, quick start, rules, and calculator pages to support future growth.

Stage 4: Ongoing Revision

Future additions may include FAQ entries, edge cases, sample games, odds variants, and a more complete calculator.

维护与更新

  • 站点基于 GitHub Pages 静态托管，便于公开访问与低成本维护。
  • 规则文本、页面结构与脚本逻辑可以独立更新，避免单个文件过度膨胀。
  • 后续更新建议统一记录在 changelog.html 与仓库提交记录中。
  • 如未来加入 logo、FAQ 体系或更完整的结算逻辑，当前目录结构可直接承接。

Maintenance & Updates

  • The site is hosted with GitHub Pages for low-cost public access and simple maintenance.
  • Rule text, page structure, and script logic can evolve independently without overloading a single file.
  • Future changes should be recorded in changelog.html and the repository history.
  • If a logo system, FAQ structure, or more advanced calculator is added later, the current layout can absorb it cleanly.
